## Key Takeaways / 重點筆記

### Module 1.5 — LLM 是什麼？

**核心本質**
- LLM 的本質是：**一台預測下一個 token 的機器**
- 它不是真的「理解」語言，而是根據**機率分布**產生最合適的下一個 token
- 好的比喻：**文字接龍**——每次都從所有可能的下一個字中，挑出機率最高的那個

**為什麼這很重要？**
- 你給的 prompt 越清楚，模型預測出「好答案」的機率就越高
- LLM 沒有真正的「理解」，所以它會在沒有明確指引時「猜錯方向」
- Agent Manager 的工作，本質上是讓 LLM 的機率分布對準你要的結果

---

### 訓練三階段

| 階段 | 說明 | 目的 |
|------|------|------|
| **Pre-training** | 餵大量文字資料，讓模型練習文字接龍 | 學習語言結構與世界知識 |
| **Fine-tuning** | 用特定對話資料微調 | 讓模型學會「好好回答問題」的格式 |
| **RLHF** | 加入人類評分的獎勵機制 | 讓回答更符合人類期待與價值觀 |

**實戰意義**
- Pre-training 決定模型「知道多少」
- Fine-tuning 決定模型「怎麼說話」
- RLHF 決定模型「說不說讓你不舒服的話」
- 選模型時，這三個階段的品質都會影響你的 Agent 行為
