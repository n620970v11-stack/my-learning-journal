## 💡 核心直覺：為什麼 Web3 特別需要 AI Agent？

**「達到自動化操作」**——這是模組三的核心價值主張。

Web3 跟傳統金融有一個根本差異：

|  | 傳統金融 | Web3 |

技術特性 → 轉換成商業價值 → 才是高管聽得懂的語言。

## 3-2 AI Agent × DeFi

### 📌 什麼是 DeFi？（課堂釐清）

**DeFi \= Decentralized Finance（去中心化金融）**

把銀行的功能，用智能合約寫成程式，部署在區塊鏈上，讓任何人都能直接使用——不需要中間人。

|  | 傳統金融 CeFi | DeFi |
| :---- | :---- | :---- |
| 中間人 | 銀行、券商 | 智能合約（程式） |
| 營業時間 | 週一到週五 | 24 小時 365 天 |
| 身份驗證 | 需要 KYC | 只需要錢包地址 |
| 資產控制 | 銀行保管 | 你自己掌控私鑰 |

**DeFi 三大核心應用：**

- **DEX（去中心化交易所）**：直接跟智能合約換幣，不需要中間人撮合  
- **借貸 Lending**：抵押幣借錢或存幣賺利息  
- **流動性挖礦 Liquidity Mining**：把幣存入池子，賺取手續費回報

---

### 📌 流動性挖礦是什麼？（課堂釐清）

**流動性**：市場上有沒有足夠的買賣雙方，讓你隨時能成交。

**流動性池（Liquidity Pool）**：用戶把資金存入公共池子，別人要換幣時直接從池子換。

你存入：1 ETH \+ 3000 USDC 到 Uniswap 池子

別人用這個池子換幣，產生 0.3% 手續費

手續費按比例分給你

→ 你賺到被動收入

**流動性挖礦 vs 銀行定存：**

|  | 銀行定存 | 流動性挖礦 |
| :---- | :---- | :---- |
| 年化報酬 | 約 1–3% | 可能 5–100%+ |
| 風險 | 極低 | 中高（幣價波動、合約風險） |
| 中間人 | 銀行 | 智能合約 |

---

### 📌 抵押借貸與清算（課堂釐清）

**為什麼要抵押 ETH 借 USDC？**

你看好 ETH 長期會漲，不想賣，但現在需要流動資金。 就像「以房增貸」——把已經擁有的房子拿去抵押，借出現金，房子還是你的。

⚠️ **正確比喻是「以房增貸」，不是「買房房貸」。** 以房增貸 \= 你已有資產，拿去抵押借錢。 買房房貸 \= 你先付自備款，銀行幫你買房。

**核心邏輯：**

你要在乎的是：**ETH 未來漲的幅度，是否會超過你付的利息成本。**

你抵押 1 ETH（3000 美元），借 1500 USDC，年利率 5%

一年後需要還：1500 \+ 75 \= 1575 USDC

只要 ETH 還在 3000 美元以上，你就沒虧

如果 ETH 漲到 5000 美元，扣掉利息淨賺約 1925 美元

**什麼是清算（Liquidation）？**

你的抵押品價值跌到危險線，智能合約自動強制把你的抵押品賣掉來還債。

你抵押 1 ETH（3000 美元），借了 1500 USDC

ETH 暴跌到 1500 美元

抵押品（1500）= 借款（1500）→ 觸發清算

智能合約強制賣掉你的 ETH 還債

你的 ETH 不見了 😢

**傳統銀行 vs DeFi 清算：**

|  | 傳統銀行 | DeFi |
| :---- | :---- | :---- |
| 執行者 | 銀行人工處理 | 智能合約自動執行 |
| 速度 | 可能給幾個月緩衝 | 條件達到立刻執行，無緩衝 |
| 申訴 | 可以跟銀行談 | ❌ 程式執行，無法撤銷 |

**補充抵押品的兩種方式：**

| 方式 | 說明 |
| :---- | :---- |
| 補充更多 ETH | 增加抵押品數量，讓健康度回升 |
| 直接還一部分 USDC | 減少借款金額，讓比例變安全 |

---

### 📌 Aave 的商業模式（課堂釐清）

Aave 靠**利息利差和清算手續費**賺錢，不靠幣價漲跌。

| 收入來源 | 說明 |
| :---- | :---- |
| 借貸利差 | 借款人付利息，Aave 抽一部分，存款人拿另一部分 |
| 清算手續費 | 發生清算時，抽取一定比例的清算獎勵 |

ETH 跌價對 Aave 影響不大——跌太多反而觸發清算，Aave 賺到手續費。 真正的風險是**壞帳（Bad Debt）**：ETH 跌太快來不及清算，抵押品價值低於借款。

### AI Agent × DeFi 三大應用

| 應用 | 解決什麼問題 | Agent 的動作 |
| :---- | :---- | :---- |
| 自動化交易 | 人類跟不上市場速度 | 監控價格 → 發現套利機會 → 自動執行 |
| 流動性管理 | 報酬率每天變化難追蹤 | 掃描利率 → 比較 → 自動移動資金到最高報酬池子 |
| 風險控制 | 清算風險隨時存在 | 監控健康度 → 預警 → 自動補倉 |

**套利（Arbitrage）**：同一個幣在不同交易所價格不同，Agent 自動低買高賣賺差價。

**自動避免清算 Agent 的設計：**

| 組成 | 內容 |
| :---- | :---- |
| 工具 | 幣價 API、鏈上健康度查詢、錢包餘額查詢、智能合約執行、DEX 換幣、推播通知 |
| 自主性層級 | Level 3 有限自主：用戶預先設定規則和授權額度，Agent 在範圍內自動執行 |
| 超出額度 | 回到 Level 2，強制請用戶確認 |

人負責策略判斷，Agent 負責風險守護。

## 3-3 AI Agent × Crypto Exchanges

### AI Agent 三大應用

**① 智能下單 Smart Order Execution**

幫用戶執行複雜的下單策略，例如 TWAP（時間加權平均價格）——在一段時間內分批買入，避免大單影響市場價格。

**② 市場分析 Market Intelligence**

持續掃描所有資訊來源，過濾出跟用戶持倉相關的重要訊號，用白話文整理推送。

**③ 用戶體驗優化 UX Enhancement**

| 場景 | Agent 的角色 |
| :---- | :---- |
| 新手不懂術語 | 即時解釋「什麼是止損？什麼是槓桿？」 |
| 用戶操作失誤 | 「你確定要用市價單嗎？現在價差較大，限價單可能更划算」 |
| 個人化建議 | 根據用戶交易歷史，推薦適合的功能或策略 |
| 客服自動化 | 24 小時回答用戶問題 |

**交易所導入 AI Agent 的商業邏輯：**

| 目的 | 說明 |
| :---- | :---- |
| 留住用戶 | 功能越強大，用戶越不想換到競爭對手 |
| 提高交易量 | 智能下單讓用戶交易更頻繁 → 手續費收入增加 |
| 降低客服成本 | AI 客服取代部分真人客服 |

**PM 思維：MaiCoin 的第一個 AI 功能應該是「用戶體驗優化」**

MaiCoin 的核心用戶是台灣入門投資者，最大的流失原因是介面複雜、術語看不懂。 導入 AI 教學助理，能有效降低新手流失率，讓更多用戶成長為長期活躍用戶。 用戶體驗優化是基礎，智能下單和市場分析是進階——要先把新手留住，才有後續。

## 3-4 AI Agent × Crypto Wallet

### 📌 什麼是 Crypto Wallet？（課堂釐清）

**Crypto Wallet 不存幣，存的是私鑰（進入資產的鑰匙）。**

你的幣永遠存在區塊鏈上：

區塊鏈帳本記錄：「地址 0xABC123 擁有 1 BTC」

錢包管理的是：

私鑰 \= 你家的鑰匙

區塊鏈地址 \= 你家的門牌號碼

幣 \= 房子裡的東西

**最重要的一句話：**

**「Not your keys, not your coins」** 私鑰不在你手上，幣就不算是你的。

**兩種錢包類型：**

| 類型 | 英文 | 私鑰在哪 | 例子 |
| :---- | :---- | :---- | :---- |
| 託管錢包 | Custodial Wallet | 平台保管 | MaiCoin、幣安帳戶 |
| 自管錢包 | Non-custodial Wallet | 你自己保管 | MetaMask、Trust Wallet |

**MetaMask 補充（2026 最新）：**

MetaMask 是目前 DeFi 功能最完整的自管錢包，整合 Uniswap、Curve、1inch 等主流 DeFi 協議，支援 Bitcoin、Solana、以太坊等多鏈。 常見風險：釣魚網站、假瀏覽器擴充功能、惡意 DApp——大多數損失都是因為用戶分享了助記詞或批准了危險交易。

---

### AI Agent × Crypto Wallet 三大應用

| 應用 | 解決什麼問題 | 對用戶的價值 |
| :---- | :---- | :---- |
| 智能交易審核 | 用戶看不懂自己在簽什麼 | 用白話文解釋，防止誤操作和詐騙 |
| 資產管理自動化 | 多鏈資產難以統一管理 | 一個介面管理所有資產 |
| 詐騙偵測與安全防護 | 詐騙手法複雜難識別 | 即時風險警告，保護資產安全 |

**Wallet vs Exchange 的 AI Agent 核心差異：**

|  | Crypto Exchange | Crypto Wallet |
| :---- | :---- | :---- |
| AI Agent 核心價值 | 幫用戶賺更多 | 幫用戶不要虧損 |
| 用戶最大需求 | 更好的交易工具 | 更高的安全感 |

**PM 思維：Wallet 的第一個 AI 功能應該是「詐騙偵測」**

Crypto Wallet 的核心挑戰是建立用戶信任。 優先導入詐騙偵測功能，能在用戶第一次使用時就建立安全感。 用戶如果被詐騙，就沒有多餘的資產可以進行後續操作。

**三個功能的優先順序：**

第一優先：詐騙偵測（建立基礎信任）

第二優先：智能交易審核（用戶開始操作後需要）

第三優先：資產管理自動化（有一定經驗後才需要）

## 3-5 AI Agent × Crypto Tools

### 什麼是 Crypto Tools？

所有幫助用戶分析、監控、管理 Crypto 資產的輔助工具。

### 四大類型

**① 鏈上數據分析 On-chain Analytics**

區塊鏈是公開帳本，所有交易透明可查。AI Agent 掃描整條鏈，發現異常模式，推送有價值的洞察。

- **巨鯨（Whale）**：持有大量加密貨幣的地址，動向往往影響市場  
- 代表工具：Nansen、Dune Analytics、Glassnode

**② 詐騙偵測 Fraud Detection**

| 偵測類型 | 說明 |
| :---- | :---- |
| 合約審計 | 掃描智能合約代碼，找出漏洞和後門 |
| 地址黑名單 | 比對已知詐騙地址資料庫 |
| 異常交易偵測 | 發現不尋常的資金流動模式 |
| 釣魚網站偵測 | 識別假冒 DeFi 網站 |

- 代表工具：Chainalysis、TRM Labs、Blockaid

**③ Portfolio 管理 Portfolio Management**

自動整合多鏈、多交易所、多 DeFi 協議的資產，統一計算總資產價值、損益、報酬率。

- 代表工具：DeBank、Zapper、Zerion

**④ 市場情緒分析 Sentiment Analysis**

掃描 Twitter、Reddit、Discord、新聞媒體，分析整體情緒（看漲 Bullish / 看跌 Bearish），結合鏈上數據給出市場判斷。

### 📌 KYT：交易所最需要的工具（課堂釐清）

**洗錢在 Crypto 會發生嗎？**

會！而且非常常見。Crypto 的匿名性歷史上一直是洗錢管道之一。

**KYT（Know Your Transaction）**：追蹤每一筆交易的資金來源是否乾淨。

AI Agent 追蹤資金來源路徑

    ↓

發現這筆錢曾流經已知的駭客地址

    ↓

自動標記為可疑，通知合規團隊

各國法規要求交易所必須配合反洗錢（AML）規定。 若被發現協助洗錢，輕則鉅額罰款，重則被迫關閉。

**被駭客攻擊怎麼辦？**

歷史案例：Mt. Gox（2014，約 4.5 億美元）、FTX（2022，數十億美元） AI Agent 即時監控資金流動，發現異常立刻警報暫停——速度是關鍵。

**BD 視角：向 Crypto Exchange 推銷工具，最有價值的兩個功能：**

| 功能 | 為什麼有價值 |
| :---- | :---- |
| 詐騙偵測／KYT | 幫交易所符合法規、避免洗錢風險、防止駭客攻擊 |
| 市場情緒分析 | 提供用戶獨家洞察，增加用戶黏著度和交易頻率 |

「給回饋」是短期策略，用戶忠誠度低。 提供「別的地方沒有的工具和洞察」，才是真正的差異化。

**商業模式：**

| 模式 | 說明 | 例子 |
| :---- | :---- | :---- |
| 訂閱制 | 進階功能付月費 | Nansen Pro |
| B2B 銷售 | 賣給交易所、機構、政府 | Chainalysis |
| API 收費 | 開發者付費使用數據 | 各類鏈上數據 API |
| 免費增值 Freemium | 基本免費，進階付費 | DeBank、Zerion |

| Wallet | 詐騙偵測 | 被詐騙就沒有後續，安全是一切的前提 |
| Tools | KYT 反洗錢 | 交易所的法規合規需求，是剛性需求 |
