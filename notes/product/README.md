# Product Hunt 產品分析

每週至少拆解一個 Product Hunt 產品，從 Core Feature PM 與 Growth PM 雙視角訓練產品思維。

## 目標
- 建立「問題優先」的產品思維，跳脫功能羅列的分析方式
- 練習 PRD Problem Statement、North Star Metric、A/B test 設計等 PM 核心輸出
- 累積競品認知與市場感知，支撐未來的 side project 決策

## 分析節奏
- 頻率：每週至少 1 篇
- 每篇最少 60 分鐘
- 每季回頭對照第 7 節的預測準確度

## 命名規則
`YYYY-MM-DD-[product-name].md`
例：`2026-06-10-notion.md`
