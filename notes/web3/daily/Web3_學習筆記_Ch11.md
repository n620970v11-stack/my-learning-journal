## 核心問題

NFT 到底在解決什麼真實問題？它不只是一張 JPEG 圖片，那它的真正價值在哪裡？

---

## 核心概念一：NFT 的本質——可程式化的所有權

NFT 的本質不是「數位圖片」，而是**鏈上所有權證明**。

更精確的說法是：**可程式化的所有權（Programmable Ownership）**——你不只是擁有一個東西，你還能在所有權裡寫入規則。

### 用門票理解這個概念

```
傳統門票的問題：
黃牛高價轉售 → 主辦方無法控制
假票橫行 → 難以驗證真偽
票券轉手 → 主辦方完全不知道
藝人想分版稅 → 完全拿不到二手市場的收益

NFT 門票的解法：
每張票是一個 NFT，有唯一 ID
      ↓
轉售時，交易紀錄永久寫在鏈上
      ↓
主辦方在合約裡設定規則：
「每次二手轉售，10% 自動回到主辦方帳戶」
      ↓
假票？不可能——鏈上只有一個 ID #001，複製不了
```

### NFT 的應用範圍

| 應用 | 說明 |
|------|------|
| 數位藝術 | 最常見，但不是最重要的用途 |
| 活動門票 | 防偽、可設定轉售規則 |
| 會員資格 | 持有 NFT 才能進某個社群 |
| 遊戲道具 | 某把劍、某塊土地 |
| 現實資產憑證 | 房產、版權（還在發展中） |
| 服務授權憑證 | Agent 的使用權限（未來方向） |

## 核心概念二：現實資產上鏈的機會與挑戰

### 房地產上鏈的想像

```
傳統房地產交易：
找仲介 → 律師審查 → 地政事務所過戶
等 2-4 週、付仲介費 + 代書費 + 規費

如果房子變成 NFT：
所有權 = 一個 NFT
轉售時直接錢包轉錢包
交易紀錄永久在鏈上，任何人都能查
幾分鐘完成、手續費幾美元
```

**關於透明化防炒房：**
```
所有交易紀錄都在鏈上
誰在炒、炒了幾間、價格怎麼走，政府和大眾都看得清楚

也可以在合約裡寫規則：
「同一個地址持有超過 3 間房子，轉售稅率自動提高到 50%」
「持有不滿 2 年就轉售，自動繳交增值稅給政府」
規則寫在合約裡，自動執行，沒有人能鑽漏洞
```

### 現實面的三大挑戰

**挑戰一：法律承認問題**
```
就算你把房子做成 NFT
法院承認這個 NFT 代表房屋所有權嗎？
目前大多數國家的法律還沒有跟上
→ 鏈上說你的，但法律說不算
```

**挑戰二：現實世界的連結**
```
NFT 是鏈上的東西
房子是現實世界的東西

有人把 NFT 賣掉，但賴在房子裡不走怎麼辦？
鏈上的所有權無法自動驅逐現實世界的人
→ 還是需要法律和執法機構介入
```

**挑戰三：現有體制的阻力**
```
仲介、代書、地政事務所都靠現有流程吃飯
改革的阻力非常大
```

### 比較現實的中間路線：房地產代幣化

```
不是把整棟房子做成一個 NFT
而是把房子切成很多份 ERC-20 代幣
→ 讓更多人可以用小額資金投資房地產
→ 就像買股票一樣買「一棟房子的 1%」
```

這個方向不需要改變現有法律體系，只是在現有框架上加一層，更容易落地。

### 共同的根本問題

> **鏈上的規則只能管鏈上的行為，管不了鏈外的行為。**

這個問題在 Web3 圈有個相關概念叫做**預言機問題（Oracle Problem）**——鏈上合約不知道現實世界發生了什麼，需要有人把現實資訊送進來。（第 15 章會深入討論）

## 核心概念三：版稅機制的困境

### 理想設計

創作者發行 NFT 時，在合約裡設定：
```
「這個 NFT 每次二手轉售
自動把成交價的 10% 打回我的錢包」
```

```
傳統藝術市場：
畫家賣出一幅畫收到 1,000 美元
這幅畫之後被轉手賣了 100,000 美元
畫家：一毛錢都拿不到

NFT 版稅：
創作者發行 NFT 賣出收到 1,000 USDC
這個 NFT 被轉手賣了 100,000 USDC
創作者自動收到 10,000 USDC（10%）
```

### 現實中的問題

版稅機制不是寫在 ERC-721 標準裡的——它是寫在**交易市場的合約**裡的：

```
OpenSea 執行版稅 → 每次在 OpenSea 成交，自動扣版稅

但如果有人繞過 OpenSea，直接錢包對錢包轉帳
→ 沒有任何合約強制執行版稅
→ 創作者拿不到任何東西
```

**2022-2023 年發生的事：**
```
Blur（新的 NFT 市場）崛起，主打「零版稅」
吸引大量交易者（因為省了版稅成本）
      ↓
OpenSea 為了競爭，也讓版稅變成可選
      ↓
創作者的版稅收入大幅下降
「NFT 讓創作者持續受益」的承諾開始動搖
```

### ERC-2981 的嘗試與局限

**ERC-2981** 是版稅資訊的標準，讓 NFT 合約能宣告版稅設定：
```
ERC-2981：「這個 NFT 的版稅是 10%，打給這個地址」
市場平台：「我知道了，但我選擇不執行」
→ 標準只能宣告，沒有強制力
```

> **核心問題：** ERC-721 標準只規定「誰擁有這個 NFT」，沒有規定「轉售時要付版稅給誰」。版稅是市場平台「自願」執行的規則，不是鏈上強制的規則。

## 核心概念四：鏈上資料 vs IPFS vs Arweave

### 三種儲存方式

**第一種：中心化伺服器（風險最高）**

```
tokenURI → "https://api.somecompany.com/nft/001.json"
```

風險：
- 公司倒閉 → 伺服器關掉 → 圖片消失
- 公司可以隨時換掉伺服器上的檔案
- 你「擁有」NFT，但內容可以被別人隨意更改

> 這是「Rug Pull（地毯式詐騙）」的常見形式之一。

**第二種：IPFS（去中心化，主流選擇）**

**IPFS（InterPlanetary File System，星際檔案系統）** 用內容的哈希找檔案，而不是用伺服器地址：

```
傳統網路：「去 https://someserver.com 找這個檔案」
→ 伺服器關掉就找不到了

IPFS：「找到哈希值是 Qm...abc 的這個檔案」
→ 任何存著這個檔案的節點都能提供
→ 內容改了，哈希就變了，連結自動失效
```

> 類比：傳統網路像是「去 7-11 買東西」——7-11 關門就買不到。IPFS 像是「找到賣這個東西的任何一個人」——只要有人存著，你就能找到。

優點：去中心化、內容不可竄改
缺點：如果沒有人持續「釘住（Pin）」這個檔案，它可能慢慢消失——IPFS 不保證永久儲存。

**第三種：Arweave（永久儲存，最強但最貴）**

你付一次費用，檔案永久儲存在網路上，任何人都無法刪除。

**Arweave 怎麼保證「永久」：**

```
你付的費用分兩部分：
一部分：立刻付給現在儲存你檔案的礦工
一部分：放進「儲存捐贈基金（Storage Endowment）」
      ↓
儲存成本每年下降（硬碟越來越便宜）
基金的利息 > 儲存成本的下降速度
→ 基金永遠夠用來支付未來的儲存費用
```

> 類比：像大學設立「獎學金基金」——一次捐一大筆，讓利息永遠夠支付每年的獎學金。

**礦工為什麼願意長期儲存——Proof of Access（存取證明）：**
```
礦工要挖新區塊時
必須證明自己還存著某個「隨機指定的舊區塊」的資料
→ 把舊資料刪掉 → 沒辦法挖新區塊 → 賺不到獎勵
→ 礦工有強烈的經濟動機保存所有歷史資料
```

Arweave 的風險：網路本身若消亡、數學模型假設失效、或遭遇監管，仍有一定風險。「永久」的更準確說法是：在網路存在的前提下，經濟模型讓儲存費用永遠有著落。

**第四種：完全上鏈（最安全但最貴）**

圖片用 SVG 格式直接寫進合約程式碼裡：
```
只要以太坊還在，這個 NFT 就永遠存在
代表作品：CryptoPunks、Nouns DAO
缺點：鏈上儲存極貴，只適合簡單圖形
```

### 四種方式對比

| 儲存方式 | 去中心化 | 永久性 | 成本 | 風險 |
|---------|---------|--------|------|------|
| 中心化伺服器 | ❌ | ❌ | 低 | 公司倒閉、內容被改 |
| IPFS | ✅ | ⚠️ 不保證 | 低 | 沒人 Pin 就消失 |
| Arweave | ✅ | ✅ | 較高 | 網路消亡風險極低 |
| 完全上鏈 | ✅ | ✅ | 極高 | 幾乎沒有 |

## ✓ 學完能做到

**情境：** 朋友花 5,000 USDC 買了 NFT，tokenURI 指向 `https://api.coolnft.xyz/metadata/001`

**正確回應：**
```
「這個 NFT 的圖片存在一個普通公司的伺服器上：

第一：如果這家公司倒閉或停止維護
→ 網址失效，NFT 圖片消失
→ 你花 5,000 USDC 買的東西變成一個空連結

第二：這家公司可以隨時換掉圖片
→ 鏈上的所有權還在你手上，但內容已經變了

建議查一下這家公司還在不在、
專案有沒有長期維護的跡象。
如果沒有，這個 NFT 的長期價值是有風險的。」
```

## 與 Agent Payment 的連結

- **可程式化所有權**：NFT 把規則寫進合約自動執行的概念，跟 agent payment 的「可程式化付款規則」是同一個底層邏輯——不需要中間人審核。
- **服務授權憑證**：未來 agent 的服務使用權、付款授權，很可能用 NFT 的形式實現——持有某個 NFT 才能呼叫某個 agent 服務。
- **版稅困境的啟示**：鏈上規則管不了鏈外行為——agent payment 方案設計時也要考慮「如果有人繞過合約直接付款，規則還有效嗎？」
