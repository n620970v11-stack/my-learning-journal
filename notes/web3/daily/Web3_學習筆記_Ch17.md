# Web3 學習筆記｜第 17 章：帳戶抽象與下一代 UX

> 學習目標：能設計一個對「不懂加密」的人友善的鏈上應用
> 完成日期：2026/06

---

## 核心問題

Web3 的 UX 為什麼這麼難用？帳戶抽象能解決什麼？

---

## 核心概念一：EOA 的根本限制

**EOA（Externally Owned Account，外部擁有帳戶）** 就是你用私鑰控制的普通錢包帳戶，例如 MetaMask。

```
EOA 的規則只有一條：
「只有持有私鑰的人才能控制這個帳戶」
```

這個設計帶來四個根本限制：

```
限制一：私鑰丟了，錢就沒了
→ 沒有「忘記密碼」、沒有客服、沒有任何補救方式
→ Web3 新用戶最大的恐懼

限制二：每一筆交易都要手動簽名
→ 不能設定「每天自動付款」這類規則
→ AI agent 沒辦法自主付款，因為它沒有私鑰

限制三：只能用 ETH 付 Gas
→ 你有 USDC 但沒有 ETH，就沒辦法發交易
→ 對新用戶非常不友善

限制四：每次只能做一件事
→ 不能把「換幣 + 存款 + 付款」打包成一筆交易
→ 三個動作要發三筆交易，付三次 Gas
```

---

## 核心概念二：帳戶抽象（Account Abstraction）

**核心想法：** 讓帳戶的控制邏輯從「必須用私鑰」變成「可以用任何你定義的規則」。

主要實作方案：**ERC-4337**，2023 年 3 月正式上線。

```
EOA 的規則（固定）：
「只有持有私鑰的人才能控制」

Smart Account 的規則（你定義）：
「需要兩個裝置同時確認才能轉帳」
「每天最多花 100 USDC，超過就拒絕」
「這個 agent 只能在週一到週五付款」
「私鑰丟了，三個朋友確認就能恢復帳戶」
```

規則寫進合約裡，帳戶就按照規則自動執行——不需要你每次都手動簽名。

---

## 核心概念三：Smart Account 的四大功能

### Social Recovery（社交恢復）
解決「私鑰丟了怎麼辦」：

```
你指定 3 個「守護者」（朋友、家人、另一個裝置）
→ 私鑰丟了
→ 3 個守護者各自確認「這是本人要求恢復」
→ 帳戶控制權轉移到新的私鑰
→ 錢完全沒有損失
```

### Session Key（會話金鑰）
解決「怎麼讓 agent 自主付款但有安全限制」：

```
你給 AI agent 一把「有限制的臨時鑰匙」：
→ 只能花 USDC，不能動其他資產
→ 每天最多花 100 USDC
→ 只能付給白名單裡的地址
→ 7 天後自動失效
      ↓
Agent 在限制內自主簽名付款
→ 不需要每次都問你
→ 就算 agent 被駭，駭客也只能在限制範圍內操作
→ 你的主帳戶完全不受影響
```

### Gas 代付（Gas Sponsorship）
解決「沒有 ETH 就不能發交易」：

```
讓 dApp 幫你付 Gas
→ 你只需要有 USDC
→ 不需要先買 ETH 才能使用 Web3
```

### 批次交易（Batch Transaction）
解決「每次只能做一件事」：

```
把換幣 + 存款 + 付款打包成一筆交易
→ 只付一次 Gas
→ 三個動作一起完成
```

## 核心概念四：Agent Payment 的完整架構

有了帳戶抽象，agent payment 的完整設計可以是：

```
第一層：Smart Account（主帳戶）
→ 放 $10,000 USDC
→ 私鑰由你自己保管
→ 設定 Social Recovery 防止私鑰丟失

第二層：Session Key 給 agent
→ 每天上限 $200 USDC
→ 只能付給白名單裡的 API 服務商
→ 只能動 USDC，不能動其他資產
→ 7 天後自動失效，需要重新授權

第三層：超過限制自動拒絕
→ 規則寫在合約裡，自動執行
→ 不需要人工介入

第四層：監控與通知
→ 監聽鏈上事件（Tenderly 等工具）
→ 發現異常立刻通知你
→ 一鍵撤銷 Session Key
→ Agent 立刻失去付款能力
```

---

## ✓ 學完能做到

**設計 Agent Payment 架構：**

Agent 需要每天訂閱各種 API 服務，預算每天最多 $200 USDC，主帳戶有 $10,000 USDC：

```
✓ Smart Account 隔離主帳戶和 agent 的操作權限
✓ Session Key 設定每日上限和白名單
✓ 超限自動拒絕，不需要人工介入
✓ 鏈上監控發現異常立即通知
✓ 異常時一鍵撤銷 Session Key
```

就算 agent 完全被駭，駭客最多只能在當天的 $200 限額內操作，主帳戶的 $10,000 完全不受影響。

## 與 Agent Payment 的直接連結

這章是整個課程最直接連到 agent payment 技術基礎的一章：

```
x402 / Skyfire 的核心架構：

你的 Smart Account
      ↓ 授權
Session Key（有限制的臨時鑰匙）
      ↓ 交給
AI agent
      ↓ 在限制內自主簽名
鏈上付款（USDC on L2）
```

- **Session Key** 解決了「agent 怎麼持有和使用私鑰」的根本問題——不是給它真正的私鑰，而是給它一把有限制的臨時鑰匙
- **Gas 代付** 讓 agent 不需要持有 ETH 也能發交易
- **批次交易** 讓 agent 能一次完成多個付款動作，降低成本
- **Smart Account 的可程式規則** 是整個 agent payment 合規設計的技術基礎
