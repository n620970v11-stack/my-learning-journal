## 核心問題

去中心化金融（DeFi）怎麼在沒有銀行的情況下，做到換幣、借貸、生息這些事？

---

## 什麼是 DeFi

**DeFi（Decentralized Finance，去中心化金融）** 是把傳統金融的中間人角色，用智能合約取代：

```
傳統金融：你 ↔ 銀行 ↔ 對方
DeFi：    你 ↔ 智能合約 ↔ 對方
```

規則寫在合約裡，自動執行，不需要任何人審核或批准。

---

## 核心概念一：DEX 與 AMM

### 傳統交易所 vs DEX

傳統交易所用**訂單簿（Order Book）**撮合買賣雙方：
```
買方掛單：我願意用 900 元買
賣方掛單：我願意用 900 元賣
價格一致 → 撮合成交
```

搬到區塊鏈上的問題：每筆掛單、撤單、撮合都要付 Gas，高頻撮合根本跑不起來。

### AMM（Automated Market Maker，自動化做市商）

不需要買賣雙方互相撮合，改用**資金池**提供流動性：

```
資金池：ETH 100 顆 + USDC 200,000 個
你放進 2,000 USDC → 直接跟池子換，拿回約 1 顆 ETH
```

### 恆定乘積公式（Constant Product Formula）

```
x × y = k（永遠保持不變）

x = 池子裡的 ETH 數量
y = 池子裡的 USDC 數量
k = 固定常數
```

**計算範例：**
```
換幣前：100 ETH × 200,000 USDC = 20,000,000（k）
放進 2,000 USDC 後：
  USDC 變成 202,000
  ETH 必須變成：20,000,000 ÷ 202,000 ≈ 99.01
  拿到：100 - 99.01 ≈ 0.99 顆 ETH
```

> 一方增加，另一方勢必減少 → 池子裡的 ETH 越少，ETH 價格越貴。

**價格滑點（Slippage）**：交易量越大，價格越不利。

### 套利（Arbitrage）自動校正價格

```
外面市場 ETH = 2,100 USDC
池子 ETH = 2,200 USDC（因為大家一直買）
      ↓
套利者：外面買 ETH（2,100）→ 拿到池子賣（2,200）→ 賺 100 USDC
      ↓
套利者一直往池子賣 ETH → 池子 ETH 增加 → 價格下降
      ↓
直到池子價格跟外面市場一致
```

套利者的自利行為，反而幫池子維持正確的價格。

### 流動性提供者（Liquidity Provider，LP）

資金池的錢來自 LP：
```
你放進：1 ETH + 2,000 USDC
→ 獲得「LP 代幣」作為憑證
→ 每次有人換幣，收一小筆手續費
→ 手續費按比例分給所有 LP
```

**目前最主流的 DEX：Uniswap**（佔整個 DEX 市場約 45% 份額）

## 核心概念二：借貸協議

### 基本邏輯

```
你把 ETH 存進借貸合約（當抵押品）
      ↓
合約根據抵押品價值，讓你借出 USDC
      ↓
借出去的 USDC 開始計息（你要付利息）
      ↓
還清 USDC + 利息，拿回 ETH
```

主流借貸協議：**Aave**、**Compound**

### 為什麼需要超額抵押？

銀行可以審核信用、透過法律追討。智能合約什麼都做不到——不認識你是誰，也沒辦法告你。

解法：**超額抵押（Over-collateralization）**——只能借抵押品價值的一部分：

```
你存入：1 ETH（價值 2,000 USDC）
最多能借：1,500 USDC（抵押品價值的 75%）
```

### 超額抵押的計算

```
初始狀態：
  抵押品：1 ETH = 2,000 USDC
  借款：1,000 USDC（50%）

ETH 跌 40%：
  2,000 × (1 - 40%) = 2,000 × 0.6 = 1,200 USDC
  抵押品還值 1,200 USDC > 借款 1,000 USDC → 協議安全
```

> 對使用者：最壞情況是失去抵押品，不需要額外掏錢補差額。
> 對協議：數學保證抵押品夠多，不靠信任使用者會還錢。

### 健康因子（Health Factor）

Aave 介面上即時顯示的數字，代表你距離清算還有多遠：
- 健康因子 > 1 → 安全
- 健康因子 = 1 → 觸發清算

### 清算機制（Liquidation）

**觸發條件：** 抵押品價值跌到某個門檻（例如借款金額的 110%）

```
清算觸發的完整流程：
ETH 跌到清算門檻
      ↓
健康因子 < 1，合約開放清算
      ↓
外部「清算人（Liquidator）」介入
      ↓
清算人幫你還清借款
      ↓
清算人獲得你的 ETH 作為報酬（比市價便宜 5-10%）
      ↓
你：借款清零，ETH 沒了，手上只剩借來的 USDC
清算人：賺走清算獎勵（Liquidation Bonus）
協議：存款人的錢安全
```

> 重要：合約不是自己去賣 ETH，而是開放給外部清算人來執行。清算人因為有獎勵才願意快速行動。

> 類比：清算人像拖吊車——不是義務幫忙，而是因為有收費才願意來。

**為什麼要用人來清算而不是合約自己執行？**
```
如果只靠合約自己清算：
網路壅塞、Gas 費暴漲時 → 合約來不及執行 → 抵押品繼續跌 → 協議虧損

開放給外部清算人：
清算機會出現 → 很多人搶著來（因為有獎勵）→ 清算速度更快、更可靠
```

### 借款人為什麼要超額抵押借錢？

1. **不想賣掉 ETH**：看好長期漲，但短期需要現金
2. **稅務考量**：賣掉資產可能觸發課稅，抵押借款不算賣出

## 核心概念三：穩定幣（Stablecoin）

### 為什麼需要穩定幣

ETH、BTC 波動太大——商家無法接受、agent 無法預測預算。

```
USDC 付款：
今天 1 USDC = 1 美元
明天 1 USDC = 1 美元  ← 可預測，適合支付
```

### 穩定幣不等於完全穩定

穩定幣只是「對美元穩定」，不是對所有貨幣穩定：
```
美元貶值 → USDC 購買力跟著貶值
對台灣人：美元跌、台幣升 → 換回台幣時拿到的更少
```

### 三種類型對比

| 類型 | 代表 | 穩定靠什麼 | 最大風險 |
|------|------|-----------|---------|
| 法幣抵押 | USDC、USDT | 銀行裡的美元 | 發行商倒閉或被監管 |
| 加密資產抵押 | DAI | 超額抵押的加密資產 | 抵押品閃崩、清算失敗 |
| 演算法型 | UST（已死） | 演算法和市場信心 | 信心崩潰、死亡螺旋 |

### 法幣抵押型（USDC、USDT）
```
Circle 收到 1 美元 → 發行 1 USDC
那 1 美元放在銀行或美國國債裡
你想換回來 → 還你 1 美元
```
優點：最穩定
缺點：依賴中心化發行商

### 加密資產抵押型（DAI）
```
存入 2,000 美元的 ETH（超額抵押）
→ 系統發行 1,000 DAI（只借出 50%）
→ ETH 暴跌時自動清算維持穩定
```
優點：去中心化
缺點：資本效率低

### 演算法型（UST — 歷史警示）
不靠任何抵押品，純粹用演算法維持 1 美元。

**2022 年 UST 死亡螺旋：**
```
有人大量拋售 UST
→ UST 跌破 1 美元
→ 市場失去信心，更多人拋售
→ 演算法瘋狂印出 LUNA 來救市
→ LUNA 被稀釋暴跌
→ UST 更沒人信
→ 72 小時內從 1 美元跌到接近 0
→ 400 億美元蒸發
```

**PM 警示：** 看到「演算法型穩定幣」要立刻問：
1. 極端市場壓力下演算法怎麼運作？
2. 有沒有實際資產作為後盾？
3. 信心崩潰時用戶資金有什麼保護？

## 補充：為什麼穩定幣在顛覆跨境支付

### SWIFT 的問題

**SWIFT** 是 1973 年成立的國際銀行訊息傳遞系統。

> 重要觀念：SWIFT 不移動錢，只傳訊息。

```
你在台灣匯款給美國朋友
台灣銀行發 SWIFT 訊息給美國銀行
「請給 XXX 帳號 1,000 美元」
銀行之間再透過中間行軋平差額
```

**為什麼慢又貴：**
```
台灣小銀行 → 找不到直接連線的美國小銀行
      ↓
需要透過中間行（Correspondent Bank）轉接
台灣小銀行 → 台灣大銀行 → 美國大銀行 → 美國小銀行
      ↓
每經過一家中間行：多收一次手續費 + 多花一天時間
結果：3-5 個工作天、多層手續費
```

**為什麼找不到直接連線的銀行：**
建立代理行關係（Correspondent Banking Relationship）需要：
- 雙方互相審查、簽署協議
- 在對方銀行預存準備金
- 維護大量合規文件

對交易量小的銀行：建立成本 >> 帶來的收益，根本不划算。

加上 2008 年後反洗錢法規趨嚴，大銀行傾向「去風險化」，越來越少跟小銀行建立關係。

### 穩定幣的解法

```
傳統 SWIFT：
台灣銀行 → 訊息 → 中間行 → 訊息 → 美國銀行
等 3-5 天、付多層手續費

穩定幣：
你的錢包 → 區塊鏈 → 對方的錢包
幾分鐘、幾美分
```

區塊鏈不在乎你是哪家銀行、在哪個國家——只要有錢包地址就能收款。

### 關於 Off-ramp / On-ramp

- **On-ramp（入金）**：法幣 → 穩定幣
- **Off-ramp（出金）**：穩定幣 → 法幣

目前很多人收到穩定幣後還是需要 Off-ramp 換回法幣（台幣、港幣等）。但隨著 Visa、Stripe、Deel 等陸續支援穩定幣結算，這個需求在慢慢減少。

## ✓ 學完能做到

**情境一：** 朋友抵押 10,000 USDC 的 ETH 借出 8,000 USDC，問要不要擔心 →
> 借了抵押品的 80%，緩衝空間很薄。ETH 只要再跌 10-15% 就會觸發清算，合約直接把 ETH 賣掉還債，不會通知你。建議補充抵押品或還一部分借款，把健康因子拉高。

**情境二：** Agent payment 方案使用演算法型穩定幣 →
> 立刻追問：極端市場壓力下怎麼運作？有沒有實際資產後盾？參考 UST 崩潰案例，演算法型穩定幣在 payment 場景風險極高。

## 與 Agent Payment 的連結

- **DEX**：Agent 需要換幣時（預算是 USDC 但服務收 ETH），透過 DEX 完成。AMM 的滑點和手續費是 agent payment 的成本因素。
- **借貸協議**：Agent 的資金可以存進借貸協議生息，等待付款期間不讓資金閒置。
- **穩定幣**：Agent payment 幾乎都選 USDC（法幣抵押型）——最穩定、最被機構信任、監管最清晰。演算法型在 UST 崩潰後無人敢用於 payment。
- **SWIFT 的局限**：Agent 在全球自主付款，不可能為每個國家的代理行關係煩惱。區塊鏈天然解決這個問題——這正是 agent payment 走加密路線的根本原因。
