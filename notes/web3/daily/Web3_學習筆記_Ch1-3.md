## 第 1 章｜為什麼會有 Web3

### 核心問題

Web1 → Web2 → Web3 的演進，本質是「資料的所有權跑去哪了」的故事。

### 中心化系統的痛點

- **信任成本**：你必須相信銀行、平台、政府不會作惡  
- **單點失效**：中央伺服器掛掉，所有人都受影響  
- **審查**：平台可以封鎖你的帳號、凍結你的資產  
- **資料壟斷**：你的資料被平台拿去獲利，你沒有控制權

### Web3 在解什麼

在沒有銀行、沒有政府、沒有任何中間人的情況下，讓陌生人之間能安全地交易。

答案是：**靠數學**。

## 第 2 章｜區塊鏈是什麼

### 核心概念一：Hash Function（雜湊函數）

**直覺：** 資料的數位指紋。

- 給任何輸入（文字、交易資料、整本書），輸出一串固定長度的字符  
- 同樣的輸入，永遠得到同樣的輸出  
- **輸入改一個字，輸出就完全不同**（不是差一點，是面目全非）  
- **單向性**：給你 Hash 值，沒辦法反推回原始輸入

輸入："Jane 轉帳 100 元給 Bob"

輸出：a3f8c2d1e9b7...（64個字符）

**用途：篡改偵測（Tamper Detection）**

- 下載檔案時，對比官方公布的 Hash 值，就能確認檔案沒被動過手腳

---

### 核心概念二：區塊結構（Block Structure）

一個區塊包含：

區塊 \= 交易資料 \+ 上一個區塊的 Hash \+ Nonce \+ 時間戳記

對這整包資料跑 Hash，得到這個區塊自己的 Hash。

**關於 Nonce（隨機數）：**

- 礦工暴力試出來的數字（不是連接區塊用的）  
- 目標：找到一個 Nonce，讓整個區塊的 Hash 開頭有很多個零  
- 沒有公式可以直接算出，只能一個個試：1、2、3……

---

### 核心概念三：鏈式結構與不可篡改性

每個區塊把上一個區塊的 Hash 鎖進自己的資料裡：

區塊 1 的 Hash → 放進區塊 2 的資料 → 區塊 2 的 Hash → 放進區塊 3...

**如果有人竄改區塊 1：**

1. 區塊 1 的交易資料改變 → 區塊 1 的 Hash 變了  
2. 區塊 2 裡包含的舊 Hash 對不上了 → 必須重找區塊 2 的 Nonce  
3. 區塊 2 的 Hash 也變了 → 區塊 3 又對不上了……  
4. 一路重算到最新區塊，同時全網還在繼續往前挖

**關鍵問答：** Q：區塊二的哈希值會因為區塊一更新而改變嗎？ A：會。區塊二的 Hash 是由「交易資料 \+ 區塊一的 Hash \+ 自己的 Nonce」跑出來的，區塊一的 Hash 一改，區塊二的輸入就變了，Hash 也跟著變。

**結論：**

不是改不了，是竄改的代價太高，以至於近乎於無法竄改。

---

### 核心概念四：共識機制（Consensus Mechanism）

**問題：** 如果有人算力夠強，能從被篡改的區塊開始重算所有區塊，怎麼辦？

**直覺：** 1000 個人每人有一份帳本副本，以多數為主——少數服從多數。

**Proof of Work（PoW，工作量證明）：**

- 想新增區塊，必須先解題（找到 Nonce）  
- 解題只能靠暴力計算，耗費大量電力和算力  
- 想篡改歷史，必須重算所有區塊，速度要比全網加起來還快  
- → 需要超過全網 51% 的算力（51% 攻擊）

**Proof of Stake（PoS，權益證明）：**

以太坊在 2022 年 9 月完成切換（稱為 **The Merge，合併**），從 PoW 換成 PoS。

運作流程：

想成為驗證者（Validator）？

        ↓

先質押（Stake）32 ETH（鎖進合約，不能動）

        ↓

系統用加權隨機抽籤，從所有驗證者中抽選

（押金越多，被抽到的機率越高，但不是押越多就一定輪到）

        ↓

被抽到的驗證者（Proposer）打包一批交易，用私鑰簽名，廣播給其他驗證者

        ↓

其他驗證者投票確認（Attestation）：這個區塊合法嗎？

        ↓

超過 2/3 驗證者同意 → 區塊正式上鏈

**防作弊機制：Slashing（懲罰）**

- 被選到後試圖寫進假交易 → 系統偵測到 → 質押的 ETH 被沒收一部分或全部

**區塊結構對比：**

PoW 區塊 \= 交易資料 \+ 上一個區塊Hash \+ 時間戳記 \+ Nonce

PoS 區塊 \= 交易資料 \+ 上一個區塊Hash \+ 時間戳記 \+ 驗證者簽名

Q：PoS 的區塊資料跑 Hash 之後也會得到 Hash 值嗎？ A：對。這包東西跑過 Hash Function，就得到這個區塊的 Hash 值，然後放進下一個區塊——鏈式結構繼續延伸。差別是 PoS 的 Hash 值不需要開頭有很多個零，因為防作弊已經靠押金機制解決了。

**PoW vs PoS 完整對比：**

|  | PoW | PoS |
| :---- | :---- | :---- |
| 防作弊方式 | 作弊需要巨大算力，成本高 | 作弊會被沒收押金 |
| 決定記帳權 | 誰先解出 Nonce 誰贏 | 加權隨機抽籤 |
| 參與門檻 | 買礦機、付電費 | 質押 ETH |
| 耗能 | 極高 | 極低（省了約 99.95%） |
| 區塊結構 | 含 Nonce | 含驗證者數位簽名 |
| 上鏈條件 | Hash 開頭有足夠多的零 | 超過 2/3 驗證者投票同意 |

**Jane 的梳理：** PoW：礦工暴力試 Nonce，直到「交易資料 \+ 上一個Hash \+ 時間戳記 \+ Nonce」跑出來的 Hash 開頭有足夠多的零，才取得記帳權。沒有人事先知道答案，Hash 值是試出來之後才產生的。

PoS：驗證者質押 ETH 參與抽籤，幣越多被抽到機率越高。被抽到後用數位簽名取代 Nonce，打包區塊廣播給其他驗證者投票，超過 2/3 同意才上鏈。

**礦工／驗證者的收益：**

- 驗證交易：沒有額外獎勵（只是參賽資格審查）  
- 成功出塊：區塊獎勵（新發行的 ETH/BTC）+ Gas Fee（所有交易的手續費）

**礦工如何驗證使用者身份：**

礦工不需要知道你是誰。驗證的不是「這個人叫什麼名字」，而是「這筆交易是由持有這個公鑰對應私鑰的人所授權的」。

| 傳統世界 | Web3 |
| :---- | :---- |
| 身份 \= 身分證、護照、帳號 | 身份 \= 你持有某把私鑰 |
| 銀行驗證你是誰 | 數學驗證你有沒有授權 |
| 身份跟你這個人綁定 | 身份跟私鑰綁定 |

## 第 3 章｜密碼學最小必備知識

### 核心問題

在沒有銀行、沒有身分證的網路上，怎麼證明「這筆交易是我授權的，不是別人假冒的」？

---

### 核心概念一：非對稱加密（Asymmetric Encryption）

**直覺：信箱類比**

| 概念 | 類比 |
| :---- | :---- |
| 私鑰（Private Key） | 開信箱的鑰匙，只有你有 |
| 公鑰（Public Key） | 信箱的地址，公開給所有人 |

**關鍵特性：**

- 私鑰可以推導出公鑰  
- 公鑰**沒辦法**反推回私鑰（單向性，跟 Hash 一樣的設計邏輯）

---

### 核心概念二：數位簽名（Digital Signature）

**重要觀念：私鑰不會出現在交易裡！**

Q：交易裡面有包含私鑰嗎？ A：絕對不會。私鑰出現在交易裡，任何人都能看到，等於把鑰匙放在信封外面寄出去。

**數位簽名的產生流程：**

完整交易資料

      ↓ Hash

    摘要（中間過程，用完即丟）

      ↓ \+ 私鑰，橢圓曲線運算（secp256k1）

    數位簽名

**廣播出去的內容：**

完整交易資料（原始的，不是摘要）+ 數位簽名 \+ 公鑰

Q：打包裡面的交易資料是已經哈希過的摘要嗎？ A：不是，是完整的原始資料。摘要只是簽名過程的中間產物，礦工需要看到完整交易資料才知道要執行什麼。

**驗證流程：**

數位簽名 \+ 公鑰 → 驗證運算 → ✓ 合法 / ✗ 不合法

**為什麼無法偽造？**

如果有人想偽造簽名，他需要私鑰。就算他拿到公鑰，橢圓曲線運算是單向的，沒辦法從公鑰反推出私鑰。

---

### 核心概念三：Merkle Tree（梅克爾樹）

**問題：** 一個區塊有幾千筆交易，怎麼快速驗證其中一筆有沒有被竄改？

**直覺：猜數字遊戲**

- 笨方法：從 1 猜到 100，最壞猜 100 次  
- 聰明方法：每次猜中間值，每次砍掉一半，最多猜 7 次

**Merkle Tree 結構：**

             Merkle Root

            Hash(A+B+C+D)

            /            \\

      Hash(A+B)        Hash(C+D)

       /     \\           /     \\

   Hash(A) Hash(B)  Hash(C) Hash(D)

      |       |        |       |

     交易A   交易B    交易C   交易D

Merkle Root 存在區塊的 Header 裡。

**驗證效率：**

- 1000 筆交易，只需要檢查約 10 個 Hash 值，就能確認任何一筆有沒有被竄改  
- 不需要下載整個區塊的完整資料（輕節點的基礎）

---

### 完整交易流程（Jane 轉 1 ETH 給 Peter）

【第一步：Jane 發起交易】

Jane 的錢包自動做：

完整交易資料（Jane → Peter，1 ETH，Gas Fee...）

        ↓ Hash

      摘要（中間產物，用完即丟）

        ↓ \+ Jane 的私鑰，橢圓曲線運算

      數位簽名

        ↓ 打包

「完整交易資料 \+ 數位簽名 \+ Jane 的公鑰」

        ↓ 廣播給全網

【第二步：礦工驗證交易】

收到交易的礦工：

1\. 用 Jane 的公鑰 \+ 數位簽名 → 驗證合法 ✓

2\. 確認 Jane 餘額 ≥ 1 ETH ✓

3\. 交易進入 Mempool（等待池）

   ↑ 此時沒有額外獎勵，只是參賽資格審查

【第三步：礦工打包與挖礦】

礦工從 Mempool 撈出一批待處理交易

（包含 Jane → Peter 這筆）

        ↓ 打包成區塊

區塊內容：

  \- 這批交易資料

  \- 上一個區塊的 Hash

  \- 時間戳記

  \- Nonce（待找）

        ↓ 開始暴力試 Nonce

Hash（交易資料 \+ 上一個區塊Hash \+ 時間戳記 \+ Nonce）

        \= 開頭有很多個零？

  不是 → Nonce \+ 1，繼續試

  是   → 找到了！

【第四步：區塊上鏈】

找到 Nonce 的礦工廣播：

「我解出來了！這個區塊由我來寫」

        ↓

全網其他節點驗證（跑一次 Hash，很快）✓

        ↓

新區塊接到鏈上

  區塊 N-1 ← 區塊 N（包含 Jane→Peter）

        ↓

Jane → Peter 的交易正式完成

礦工獲得：區塊獎勵（新發行的 ETH）+ Gas Fee

**時間感：** | 階段 | 時間 | |------|------| | 第一步（簽名廣播） | 瞬間完成 | | 第二步（驗證） | 幾秒內 | | 第三步（挖礦） | 以太坊約 12 秒，比特幣約 10 分鐘 | | 第四步（確認） | 上鏈後再等幾個區塊才算絕對安全 |

## 第一部分整體串聯

- **Ch 1**：Web3 在解決「資料所有權」的問題——不讓中心化機構壟斷你的資料和資產  
- **Ch 2**：區塊鏈用 Hash \+ 鏈式結構 \+ 共識機制，做到沒有中央機構也能信任的帳本  
- **Ch 3**：密碼學提供了「身份」和「授權」的工具——私鑰證明你是你，簽名證明你授權了

## 與 Agent Payment 的連結

- **不可篡改性** → AI agent 的付款記錄可以被信任，沒有任何一方能偷偷修改記錄  
- **數位簽名** → Agent 付款時用簽名證明「這筆付款是被授權的」  
- **私鑰安全** → 「Agent 怎麼安全地持有和使用私鑰」是 Skyfire 這類方案要解決的核心問題；你不可能把私鑰直接交給 AI agent
