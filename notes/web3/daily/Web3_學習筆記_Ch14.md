## 核心問題

區塊鏈有這麼多條，它們之間怎麼溝通？資產怎麼跨鏈移動？

---

## 核心概念一：跨鏈橋（Bridge）

### 為什麼需要跨鏈橋

每條鏈（以太坊主網、Arbitrum、Solana）都是獨立的帳本，彼此之間沒有天然連結。要把資產從一條鏈搬到另一條鏈，需要一個中間機制——跨鏈橋。

### Lock & Mint / Burn & Unlock

跨鏈橋的基本運作方式：

```
從以太坊主網 → Arbitrum：

步驟一：把 USDC 存入跨鏈橋合約
→ 主網上的 USDC 被「鎖住」（Lock）
→ 合約持有這筆錢

步驟二：Arbitrum 上「鑄造」等值的 USDC
→ 這是全新產生的，背後有主網鎖住的 USDC 作擔保

反向（Arbitrum → 主網）：
→ Arbitrum 上的 USDC 被「銷毀」（Burn）
→ 主網上鎖住的 USDC 被「釋放」（Unlock）給你
```

### 為什麼是鎖住而不是銷毀？

USDC 的 `burn()` 功能只有 Circle 有權限呼叫。跨鏈橋合約沒有這個權限，所以只能把 USDC 轉移到合約地址鎖住——效果上等同於「暫時消失」，但實際上錢還在合約裡。

> **門檻設計（Threshold）** 是開發團隊在部署合約時寫進程式碼的規則，規定需要幾個驗證節點同意交易才算有效。這個規則公開透明、寫在合約裡，任何人可以去 Etherscan 查——官網可以說謊，合約說不了謊。

## 核心概念二：為什麼跨鏈橋是駭客最愛的目標

**核心原因：資產集中在單一合約。**

```
正常合約：用戶資產分散在幾千個地址
→ 攻破一個地址，只拿走那個人的錢

跨鏈橋合約：所有過橋的資產鎖在同一個地址
→ 攻破一個合約，拿走所有人的錢
```

TVL（Total Value Locked，總鎖倉量）越高，對駭客的吸引力越大。**TVL 高代表受歡迎，不代表安全。**

### 歷史上的重大事故

| 事故 | 年份 | 損失 |
|------|------|------|
| Ronin Bridge | 2022 | $6.25 億 |
| Wormhole Bridge | 2022 | $3.2 億 |
| Nomad Bridge | 2022 | $1.9 億 |
| Harmony Bridge | 2022 | $1 億 |

### Wormhole 攻擊方式拆解

```
正常流程（以太坊 → Solana）：
在以太坊鎖住 ETH → 合約確認 → Solana 鑄造 wrapped ETH

攻擊流程：
跳過鎖住步驟
→ 偽造「已在以太坊鎖了 120,000 ETH」的假證明
→ 合約驗證邏輯有漏洞，接受了假證明
→ 在 Solana 上鑄造 120,000 wrapped ETH 給攻擊者
→ 攻擊者再用 wrapped ETH 換回以太坊上的真實 ETH
→ 拿走其他用戶鎖在合約裡的錢
```

攻擊者拿走的是**其他用戶**鎖在合約裡的錢，自己一毛錢都沒存。

### Ronin Bridge 攻擊方式

```
Ronin 有 9 個驗證節點，需要 5 個同意才能通過
→ 攻擊者控制了其中 5 個節點
→ 直接通過偽造的提款交易
→ 拿走所有鎖倉資產
```

## 核心概念三：三種驗證方式

跨鏈橋需要確認「另一條鏈上的事件是真的」，有三種方式：

| 方式 | 原理 | 優點 | 缺點 |
|------|------|------|------|
| 見證人節點 | 一組節點投票確認事件 | 容易實作 | 節點被收買或駭客控制就完了 |
| Light Client | 在目標鏈上直接用密碼學驗證來源鏈資料 | 安全性高 | 技術複雜、Gas 費用高 |
| ZK 證明 | 跨鏈事件產生數學證明，目標鏈驗證 | 最安全 | 工程難度極高，還在發展中 |

**現實：** 大多數橋用見證人節點，因為最容易實作——但也是被駭最多的設計。

## 核心概念四：訊息傳遞協議

跨鏈橋只能搬資產，**訊息傳遞協議（Cross-chain Messaging Protocol）** 能傳任何東西——資產、指令、資料、任何合約呼叫。

### LayerZero

用兩個互相獨立的角色互相把關：

```
Oracle（預言機）：把來源鏈的「區塊頭」傳到目標鏈
Relayer（中繼者）：把「這筆交易存在的證明」傳到目標鏈

兩者必須同時同意，訊息才算有效
→ 就算其中一個被駭，另一個還能把關
→ 要同時收買兩個互不相識的角色才能偽造
```

### CCIP（Chainlink 跨鏈互通協議）

Chainlink 推出，設計更保守：

```
額外加一層「風險管理網路」
→ 獨立監控所有跨鏈訊息
→ 發現異常可以緊急暫停
```

Circle 的官方 USDC 跨鏈方案（CCTP）與 Chainlink 合作，是目前最被機構信任的跨鏈 USDC 方式。

## 補充：預言機（Oracle）

智能合約是封閉的，只能讀取鏈上資料，無法自己查詢鏈外世界（ETH 價格、天氣、選舉結果）。

**預言機的功能：把「鏈外資料」餵進鏈上。**

```
現實世界（ETH 價格 = $3,000）
      ↓
預言機把資料寫進鏈上
      ↓
智能合約讀取並使用
```

幾乎所有需要現實世界資料的合約都會接預言機。Chainlink 是目前最大的預言機服務。

### 預言機攻擊（Oracle Manipulation）

```
攻擊者在小型 DEX 上用大量資金砸低 ETH 價格
→ 預言機讀到假價格
→ 合約觸發大量錯誤清算
→ 攻擊者趁機獲利
```

防禦方式：用多個預言機來源取平均值，而不是只接一個。

## ✓ 學完能做到：評估跨鏈橋安不安全

看到「15 個驗證節點、TVL 10 億美元」，應該追問：

```
1. 門檻是多少？需要幾個節點同意才能通過？
   → 門檻越高越安全，但也越慢

2. 這 15 個節點是誰在跑？夠不夠獨立分散？
   → 如果都是同一家公司的伺服器
   → 攻破一個目標 = 控制全部節點

3. 有沒有安全審計報告？
   → 第三方審計是基本要求

4. 有沒有緊急暫停機制？
   → 出事能不能即時止損

5. 去 Etherscan 看合約原始碼
   → 確認門檻設計是否跟對外宣稱的一致
   → 官網可以說謊，合約說不了謊
```

**TVL 高只代表受歡迎，保險箱裡的錢越多，只是讓它變成更大的攻擊目標。**

## 與 Agent Payment 的連結

- **多鏈付款需求：** Agent 的預算可能在主網，服務部署在不同的 L2——靠跨鏈橋一筆筆搬錢慢又有風險，訊息傳遞協議讓 agent 在一條鏈發出指令、自動在其他鏈執行付款
- **USDC 跨鏈：** Circle 官方的 CCTP（配合 CCIP）是最被機構信任的 USDC 跨鏈方案——agent payment 幾乎都用 USDC，這個選擇直接影響方案的安全性和合規性
- **預言機與換幣：** Agent 需要根據即時匯率換幣付款時，依賴預言機提供價格資料——預言機的可靠性直接影響付款的正確性
