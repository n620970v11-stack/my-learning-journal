# Web3 學習筆記｜第 12 章：DAO 與鏈上治理

> 學習目標：能參與一個 DAO 的治理，也看得出哪些 DAO 是真的、哪些是裝飾
> 完成日期：2026/06

---

## 核心問題

如果沒有公司、沒有 CEO、沒有董事會，一個組織怎麼做決策、怎麼管理資金、怎麼執行決定？

---

## 核心概念一：什麼是 DAO

**DAO（Decentralized Autonomous Organization，去中心化自治組織）** 是一種用智能合約管理規則和資金的組織形式。

三個字拆開來理解：
```
Decentralized（去中心化）：沒有單一的控制者
Autonomous（自治）：規則自動執行，不需要人工審核
Organization（組織）：有共同目標、有資金、有成員
```

### DAO vs 傳統股份公司

| | 傳統股份公司 | DAO |
|--|------------|-----|
| 投票權來源 | 持有股份 | 持有治理代幣 |
| 規則載體 | 公司章程 + 法律體系 | 智能合約 |
| 執行方式 | 需要人工執行 | 自動執行 |
| 財庫透明度 | 不公開 | 完全公開 |
| 信任基礎 | 法律和信任 | 數學和經濟激勵 |

## 核心概念二：DAO 的現實問題

理想的 DAO 與現實往往有落差：

```
理想的 DAO：
所有人平等參與、民主決策、完全透明

現實的 DAO：
少數核心貢獻者推動大多數事情
大多數代幣持有者根本不參與
幾個大戶控制關鍵投票
```

### 問題一：投票率極低
```
很多 DAO 的投票率不到 5-10%

原因：
- 有些人買代幣只是為了投機，不關心治理
- 投票本身要付 Gas
- 提案太多太複雜，懶得看

結果：少數人實際上控制了「去中心化」組織的決策
```

### 問題二：大戶壟斷（鯨魚問題）
```
治理代幣的投票權跟持有量成正比
→ 買越多代幣，投票權越大
→ 有錢的人（鯨魚）可以買下大量代幣
→ 直接控制投票結果
→ 跟「去中心化」的初衷完全相反
```

### 問題三：執行速度太慢
```
傳統公司：CEO 今天決定，明天執行

DAO：
發起提案 → 討論期（3-7 天）
→ 投票期（3-7 天）
→ 時間鎖（2-3 天）
→ 才能執行
整個流程可能要 2-3 週
```

### 問題四：資訊不對稱
```
鏈上投票結果透明
但投票前的討論大多在鏈下（Discord、論壇、Twitter）
→ 充滿雜訊、假資訊、操縱
→ 很難確認哪些資訊是真的
→ 最會包裝資訊的人可能贏得投票，不是最正確的決策
```

### DAO 真正的優勢：財庫管理
```
傳統組織的錢放在銀行帳戶
→ 需要信任財務人員不會捲款
→ 銀行可以凍結帳戶

DAO 的財庫（Treasury）放在智能合約裡
→ 任何人都能看到餘額
→ 沒有提案和投票通過，沒有人能動這筆錢
→ 完全透明、完全自動執行
```

目前最大的幾個 DAO 財庫管理著數十億美元。

## 核心概念三：多簽錢包（Multisig Wallet）

### 基本概念

需要多個人同時授權，才能執行一筆交易。

> 類比：電影裡銀行金庫需要兩把鑰匙同時轉動才能打開——一把在行長手上，另一把在副行長手上。

**2-of-3 多簽範例：**
```
設定：3 個人裡面，需要至少 2 個人簽名才能動用資金

要轉出 100 ETH：
Person A 簽名 ✅
Person B 簽名 ✅
Person C 還沒簽
→ 2 個簽名已達門檻 → 交易執行
```

### 多簽解決的兩個根本問題

**問題一：單點失效**
```
只有一個人管錢：
→ 私鑰丟失 → 錢永久消失
→ 被駭客攻擊 → 錢全部被盜
→ 想捲款跑路 → 沒有任何阻止機制

多簽：
→ 一個人的私鑰丟失 → 其他人還能救回來
→ 一個人想捲款 → 其他人可以否決
```

**問題二：需要信任單一個人**
```
傳統組織：你必須相信財務長不會捲款

多簽 DAO：
你不需要相信任何單一個人
只需要相信「3 個人裡不會有 2 個人同時作惡」
→ 把信任分散，大幅降低風險
```

### 多簽在緊急情況的應用
```
協議被駭客攻擊，需要立刻暫停合約
如果等 DAO 投票：2-3 週 → 錢早被盜光了

解法：
DAO 授權「緊急委員會」，由核心成員組成 3-of-5 多簽
緊急情況下 3 個人同意就能立刻執行，不需要等全體投票
```

### 最主流的多簽工具：Safe（前身 Gnosis Safe）
- ERC-4337 相容的智能合約錢包
- 可設定任意的 m-of-n 簽名門檻
- 目前很多大型 DAO 的財庫都放在 Safe 裡管理

### 多簽成員的重要性
```
3-of-5 多簽聽起來不錯
但要問：「這 5 個人是誰？」

如果 5 個人都是同一個公司的員工
→ 公司出問題，5 個人可能同時出問題
→ 多簽的保護形同虛設

理想的多簽：
5 個人來自不同背景、不同地區
彼此之間沒有利益關係
→ 很難同時被收買或脅迫
```

## 核心概念四：鏈上治理 vs 鏈下治理

### 鏈上治理
```
有人發起提案
→ 持幣者在鏈上投票（每一票都是一筆交易）
→ 投票結果直接觸發智能合約執行
→ 通過了 → 合約自動執行決議
→ 否決了 → 什麼都不發生
```

**問題：每一票要付 Gas**
```
10,000 個持幣者投票
每筆 Gas 費 5 美元
→ 全體投票一次要花 50,000 美元
→ 很多人直接放棄投票 → 投票率更低
```

### 鏈下治理（Snapshot）

**Snapshot** 是目前最主流的鏈下治理平台：

```
在某個區塊高度「拍快照」
記錄每個地址當時持有多少治理代幣
      ↓
持幣者用私鑰簽名表達意見
（簽名不需要廣播到鏈上，不付 Gas）
      ↓
Snapshot 平台統計結果
      ↓
核心團隊或多簽委員會根據結果執行
```

**免費，但有代價：**
```
鏈上治理：結果自動執行，不需要信任任何人
鏈下治理：結果需要人工執行，必須相信核心團隊會照結果做
```

### 兩種方式對比

| | 鏈上治理 | 鏈下治理（Snapshot） |
|--|---------|-------------------|
| 費用 | 每票付 Gas | 免費 |
| 執行 | 自動執行 | 需要人工執行 |
| 信任需求 | 不需要信任任何人 | 需要信任核心團隊 |
| 速度 | 較慢 | 較快 |
| 去中心化程度 | 高 | 較低 |

### 現實中大多數 DAO 的做法：混合使用

```
重大決策（改變合約規則、動用大量資金）
→ 鏈上治理，確保完全去中心化

日常決策（社群方向、小額支出、合作提案）
→ Snapshot 鏈下投票，省錢省時間
→ 核心團隊根據結果執行
```

## 核心概念五：治理攻擊與時間鎖

### 治理攻擊（Governance Attack）

```
攻擊者借入大量治理代幣（透過 DeFi 借貸協議）
      ↓
在同一筆交易裡：
借代幣 → 用代幣投票通過惡意提案 → 執行提案 → 還代幣
（這叫做閃電貸攻擊，Flash Loan Attack）
      ↓
整個過程在一個區塊內完成
攻擊者可能把 DAO 財庫的錢全部轉走
```

### 時間鎖（Timelock）

```
提案通過後，必須等 48-72 小時才能執行
→ 讓社群有時間發現惡意提案並緊急取消
→ 防止治理攻擊
```

## ✓ 評估一個 DAO 的四個關鍵問題

看到一個 DAO，要問：

```
1. 代幣分配夠不夠分散？
   → 前 10 個地址持有多少比例？
   → 防止大戶壟斷投票

2. 執行機制是鏈上還是鏈下？
   → 鏈下需要信任核心團隊
   → 核心團隊是誰？怎麼產生？能被替換嗎？

3. 多簽的成員是誰？夠不夠獨立？
   → 成員背景是否多元、來自不同組織
   → 防止單點失效

4. 有沒有時間鎖？等待期多長？
   → 防止治理攻擊
   → 通常 48-72 小時是合理範圍
```

## 補充：DAO 的工作機會

DAO 相關職位正在快速專業化，對 PM / Strategy 背景最相關的職位：

| 職位 | 內容 | 薪資參考 |
|------|------|---------|
| 治理分析師 | 撰寫提案、分析投票趨勢、設計治理機制 | $60k-$120k + 代幣 |
| 財庫管理 | 管理多簽財庫、資產配置、風險管理 | $60k-$250k + 代幣 |
| 社群經理 | 協調貢獻者、社群建設、治理參與 | $50k-$100k + 代幣 |
| AI 治理策略師 | 設計和管理 DAO 裡的 AI agent（新興角色） | 還在發展中 |

**新興趨勢：** DAO 正在實驗用 AI agent 做提案摘要、輿情分析、自動化投票委託——這正是 agent payment + Web3 治理的交叉點。

**入門路徑：**
```
參與真實的 DAO（Uniswap、Aave、Compound 的論壇和投票）
→ 累積鏈上治理參與紀錄
→ 貢獻提案或研究報告
→ 建立在 Web3 社群的聲譽
→ 申請正式職位
```

## 與 Agent Payment 的連結

- **多簽 + Agent**：agent 的簽名加上你的預授權才能動用資金，就算 agent 被駭也無法單獨轉走——多簽概念直接應用在 agent payment 的安全設計。
- **DAO 財庫管理**：agent payment 方案管理用戶資金的設計，參考的正是 DAO 財庫的透明化和多簽架構。
- **AI 治理**：DAO 用 AI agent 做提案摘要和投票委託，是 agent payment 在治理場景的延伸應用。
