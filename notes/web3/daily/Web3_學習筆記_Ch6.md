## 核心問題

以太坊跟比特幣有什麼根本上的不同？

---

## 核心概念一：以太坊是什麼

**比特幣 vs 以太坊：**

| | 比特幣 | 以太坊 |
|--|--------|--------|
| 功能 | 只能轉帳 | 可以跑任何程式 |
| 類比 | 計算機（只做一件事）| 電腦（可以跑任何程式）|
| 設計哲學 | 簡單才安全 | 可程式化的世界電腦 |

以太坊的創辦人 Vitalik Buterin 在 19 歲時提出：
> 「如果有一台去中心化的電腦，任何人都能在上面部署程式，而這些程式的執行結果沒有任何人能篡改或阻止，會發生什麼事？」

在以太坊上跑的程式，就叫做**智能合約（Smart Contract）**。

---

## 核心概念二：Gas（燃料費）

**為什麼需要 Gas？**

> Q：什麼叫每個人都共用一台電腦？是只有一個伺服器嗎？
> A：不是一台伺服器，是全世界幾千台電腦同時在跑同一件事。這幾千台電腦各自獨立運作，但每一台上面的資料完全一樣，任何一筆交易這幾千台都要執行一遍確認結果一致。「共用」的意思是這幾千台電腦的計算能力有限，所有人的交易都在搶這條有限的高速公路。

如果有人部署了無限迴圈的程式，這台世界電腦會被卡死。

**Gas 的解法：**
> 你要這台電腦幫你執行程式？可以，但你要為每一個運算步驟付費。

```
每執行一個加法    → 付 3 Gas
每讀一次資料     → 付 200 Gas
每寫一次資料     → 付 20,000 Gas
```

無限迴圈每跑一圈都消耗 Gas，Gas 用完強制停止，不會卡死網路——只會把發起者的 Gas 燒光。

**Gas 的計價單位：Gwei**
```
1 ETH = 1,000,000,000 Gwei（10億 Gwei）
1 Gwei = 0.000000001 ETH
```

> Q：Gas Fee 是 by 平台的，可以用 BTC 計價嗎？
> A：不行。每條鏈只能用自己的原生代幣付 Gas：以太坊用 ETH、BNB Chain 用 BNB、Solana 用 SOL。BTC 是比特幣網路上的資產，以太坊根本不認識它。就算有 WBTC（在以太坊上發行的代幣），也不能用來付 Gas。

**完整 Gas 費用估算範例（Jane 轉 100 USDC 給 Peter）：**
```
讀取 Jane 的餘額        → 200 Gas
驗證簽名               → 3,000 Gas
更新 Jane 的餘額        → 20,000 Gas
更新 Peter 的餘額       → 20,000 Gas
記錄這筆交易            → 5,000 Gas
                        ──────────
總共需要約              → 48,200 Gas

Gas Limit 設定：60,000（最多願意用這麼多）
Gas Price：20 Gwei（每單位願意付 20 Gwei）
最多花費：60,000 × 20 Gwei = 0.0012 ETH
實際只用 48,200 Gas，剩下退還
```

## 核心概念三：EIP-1559 費用機制

**EIP-1559 之前的問題：**
Gas 費用完全靠喊價競爭，不知道要出多少錢才能讓交易被處理。

**EIP-1559 之後：**
```
你付的 Gas Fee = Base Fee + Priority Fee
```

| 費用組成 | 說明 | 去哪裡 |
|---------|------|--------|
| Base Fee（基本費）| 系統自動計算，根據網路壅塞程度調整 | 🔥 直接銷毀 |
| Priority Fee（小費）| 你額外給驗證者的小費，讓他優先處理你的交易 | 💰 驗證者 |
| 區塊獎勵 | 系統每個區塊新發行的 ETH | 💰 驗證者 |

**為什麼 Base Fee 要銷毀？**

> Q：ETH 被銷毀是什麼意思？
> A：把 ETH 永久轉到一個沒有人持有私鑰的地址（Burn Address：0x000...0000），裡面的 ETH 永遠無法被取出，等於從世界上消失了。

> Q：銷毀的目的是為了讓供需平衡嗎？
> A：對。設計目的有兩個：
> 1. **讓費用機制更可預測**：如果 Base Fee 給礦工，礦工有動機人為製造壅塞拉高費用；銷毀掉就斷了這個動機。
> 2. **供需平衡**：網路越忙 → 銷毀越多 → ETH 供應量減少 → 稀缺性增加。

```
網路使用量高 → 銷毀多 → ETH 變稀缺 → 價值支撐
網路使用量低 → 銷毀少 → ETH 供應正常
```

以太坊自 EIP-1559 上線以來，已累積銷毀超過 400 萬顆 ETH。

**怎麼知道小費要給多少？**

錢包軟體會自動估算，給你幾個選項：
```
🐢 慢速：Base Fee + 1 Gwei　→　便宜，可能等 5-10 分鐘
🚗 標準：Base Fee + 1.5 Gwei　→　約 15-30 秒
🚀 快速：Base Fee + 2 Gwei　→　下一個區塊就處理
```

Gas 費用較低的時機：平常半夜（台灣時間），網路較閒。
Gas 費用暴漲的時機：熱門 NFT mint、重大市場波動。

## 核心概念四：兩種帳戶

**以太坊有兩種帳戶：**

| | EOA（Externally Owned Account，外部帳戶）| Contract Account（合約帳戶）|
|--|----------------------------------------|---------------------------|
| 控制方式 | 私鑰 | 程式碼（規則）|
| 主動性 | 可以主動發起交易 | 只能被觸發後執行 |
| 有無私鑰 | 有 | 沒有 |
| 信任基礎 | 信任私鑰持有者 | 信任程式碼邏輯 |
| 例子 | 你的 MetaMask 錢包 | Uniswap、Aave、NFT 合約 |

**類比：**
- EOA 像一個**人**——可以主動走進銀行說「我要轉帳」
- Contract Account 像一台**自動販賣機**——不會主動做任何事，但你投幣（觸發），就按設定好的規則執行

**互動方式：**
```
EOA（你）→ 發起交易 → 觸發 Contract Account → 合約自動執行
```

沒有 EOA 主動觸發，合約永遠不會自己動。

> Q：Contract Account 是誰制定的？
> A：開發者用 Solidity 寫程式碼，部署到以太坊上之後就沒有人能修改——連開發者自己也改不了（除非事先在合約裡加了「可升級」的機制）。「Code is Law（程式碼即法律）」。

> Q：我要怎麼知道有這個交易或服務可以使用？
> A：大部分人透過前端網站使用（Uniswap、Aave、OpenSea），網站幫你組好交易送到合約。如果想直接查合約，可以去 Etherscan 用合約地址查詢。安全提醒：一定要從官方網站確認合約地址，不要點來路不明的連結。

**為什麼說「智能合約讓以太坊從轉帳工具變成世界電腦」？**
> 因為不只有轉帳功能，以太坊還包括許多其他交易功能。任何人都可以在上面部署任何規則，而且這些規則自動執行、沒有人能阻止、沒有人能篡改。

## 核心概念五：Nonce（交易計數器）

**三個地方都叫 Nonce，意思完全不同：**

| 出現位置 | Nonce 的意思 |
|---------|------------|
| PoW 挖礦 | 礦工暴力試出來的隨機數，讓 Hash 開頭有很多個零 |
| SIWE 登入 | 伺服器產生的一次性隨機數，防止重放攻擊 |
| EOA 交易 | 帳戶的交易計數器，從 0 開始每筆交易加 1 |

**EOA 交易 Nonce 的兩個功能：**

| 功能 | 說明 |
|------|------|
| 防止重複執行 | 每個 Nonce 只能用一次，舊 Nonce 重放會被拒絕 |
| 強制順序執行 | Nonce 必須連續，不能跳號 |

```
你的帳戶 Nonce = 5

第一筆交易：Nonce = 6
第二筆交易：Nonce = 7

→ Nonce 7 必須等 Nonce 6 先完成才能執行
```

## 交易狀態與 Mempool

**交易的三個階段：**
```
你發出交易 → 進入 Mempool（等待室）→ 被打包進區塊 → 上鏈確認
                    ↑
              還在這裡就可以覆蓋
```

Mempool 是「等待室」，不是「已確認」。只有上鏈之後才是不可逆的。

**覆蓋卡住的交易（Replace by Fee，RBF）：**

> Q：可以重新發交易嗎？即使這筆交易已經被放到 Mempool？
> A：可以！只要交易還在 Mempool、還沒被打包進區塊，就可以覆蓋。

| 條件 | 可以覆蓋嗎 |
|------|-----------|
| 交易在 Mempool 等待中 | ✅ 用同樣 Nonce + 更高小費覆蓋 |
| 交易已被打包進區塊 | ❌ 已上鏈，不可逆 |

**情境：兩筆交易都卡住怎麼解決？**

```
問題：
第一筆交易：Nonce = 6，Priority Fee 很低 → 卡在 Mempool
第二筆交易：Nonce = 7 → 必須等 Nonce 6，也卡住了

解法：
重新發 Nonce = 6 的交易，Priority Fee 拉高
→ 驗證者選小費高的那筆執行，舊的 Nonce = 6 自動作廢
→ Nonce = 6 完成後，Nonce = 7 也能繼續執行
```

> 這個操作叫做 **Replace by Fee（RBF）**，MetaMask 的「加速交易」功能本質上就是在做這件事。

## 與 Agent Payment 的連結

- **Gas 機制**：Agent 替你付款時，每一筆鏈上交易都需要付 Gas。這是為什麼 agent payment 方案需要管理 ETH 餘額，不只是管理付款用的穩定幣。
- **Contract Account**：Skyfire 這類 agent payment 方案，核心就是部署一個智能合約來管理 agent 的付款權限——用程式碼定義 agent 能做什麼、不能做什麼，而不是靠人工審核。
- **Nonce 管理**：Agent 自動發出大量交易時，Nonce 的順序管理是一個實際的工程問題——如果多個 agent 共用同一個帳戶，Nonce 衝突會讓交易全部卡住。
