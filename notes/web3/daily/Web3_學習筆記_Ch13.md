# Web3 學習筆記｜第 13 章：Layer 2 與擴容方案

> 學習目標：能說明選擇 L2 部署時要考量什麼
> 完成日期：2026/06

---

## 核心問題

以太坊明明這麼強，為什麼還需要 L2？

---

## 核心概念一：不可能三角（Blockchain Trilemma）

區塊鏈設計有三個目標，三者只能同時兼顧其中兩個：

```
        去中心化
       （Decentralization）
            △
           / \
          /   \
         /     \
安全性 ————————— 擴展性
（Security）   （Scalability）
```

| 選擇 | 犧牲了什麼 | 例子 |
|------|-----------|------|
| 去中心化 + 安全，但慢 | 擴展性 | 以太坊主網 |
| 去中心化 + 快，但不安全 | 安全性 | 某些小鏈 |
| 快 + 安全，但中心化 | 去中心化 | Visa、支付寶 |

**以太坊的選擇：** 主網只做「去中心化 + 安全」，把速度問題交給 L2 解決。

---

## 核心概念二：L2 是什麼

**類比：** 以太坊主網是公證處，L2 是公證處外面的便民服務站。

```
傳統（沒有 L2）：
每一筆交易直接在以太坊主網執行
→ 幾千台節點各自驗證 → 慢、貴

L2 的做法：
1000 筆交易先在 L2 上執行（鏈外）
→ 打包成一個「摘要」
→ 只把這個摘要提交到以太坊主網
→ 費用由 1000 筆交易分攤
```

**L2 也是一條真實的鏈**，有自己的區塊、節點、出塊機制——只是定期把摘要錨定到以太坊主網做最終結算。

**繼承安全性（Inherited Security）：** 就算 L2 整個出問題，你的資金仍然安全，因為最終狀態錨定在以太坊主網上，永遠可以繞過 L2 把錢取回來。

### 補充：什麼是「最終結算」（Final Settlement）

類比：你用 Line Pay 在便利商店買咖啡，刷完當下交易「完成」了——但這筆錢還沒有真正從你的銀行帳戶轉到便利商店的帳戶。Line Pay 和銀行每天會在某個時間點做一次「軋帳」，把今天所有交易一次結清，錢才真正動起來。這個動作就叫**結算**。

在 L2 的脈絡裡：

```
你在 Arbitrum 上轉帳
→ Arbitrum 帳本立刻更新（幾秒內）
  但只有 Arbitrum 自己知道
      ↓
過一段時間，摘要提交到以太坊主網
→ 主網正式記錄 → 這才是「最終結算」
  全網永久記錄，不可更改
```

在主網完成最終結算之前，理論上帳本還有被竄改的可能。一旦提交到以太坊主網，就跟所有鏈上資料一樣——改不掉、刪不掉、永久存在。這就是「安全性最終由以太坊主網保障」的具體意思。

## 核心概念三：兩種主流 L2 設計

### Optimistic Rollup（樂觀型卷軸）

**核心邏輯：** 預設相信提交的結果是誠實的，給一段時間讓任何人挑戰。

```
L2 把執行結果提交到主網
→ 進入「挑戰期」（通常 7 天）
→ 沒人挑戰 → 自動確認
→ 有人挑戰 → 主網重新驗證那筆有爭議的交易
```

**經濟設計：**
- 挑戰者發現作弊 → 挑戰者獲得獎勵
- 提交者被抓到作弊 → 押金被沒收（Slash）

**最大缺點：** 提款回主網需要等 7 天。

**代表：** Arbitrum、Optimism（OP Mainnet）

---

### ZK Rollup（零知識卷軸）

**核心邏輯：** 提交數學證明（Zero-Knowledge Proof），主網直接驗證，不需要等待期。

```
L2 處理 1000 筆交易
→ 產生一份數學證明（證明計算過程完全正確）
→ 提交到以太坊主網驗證
→ 證明合法 → 立刻確認
→ 不需要 7 天等待
```

**優點：** 提款快（幾分鐘）、數學保證安全
**缺點：** 產生證明非常耗計算資源，工程複雜度高

**代表：** zkSync、Starknet、Linea

---

### 兩種方案對比

|  | Optimistic Rollup | ZK Rollup |
|--|-------------------|-----------|
| 提交內容 | 執行結果 | 數學證明 |
| 主網如何信任 | 樂觀相信 + 挑戰期 | 直接驗證證明 |
| 提款等待時間 | 7 天 | 幾分鐘 |
| 技術成熟度 | 成熟、生態大 | 長期潛力更大，但工程複雜 |
| 代表 | Arbitrum、Optimism | zkSync、Starknet |

> 業界普遍認為長期 ZK Rollup 會是主流，但 Optimistic Rollup 在過渡期仍是主力。

## 核心概念四：L2 費用結構

你付給 L2 的總費用 = **L2 自己的執行費** + **分攤到你身上的主網費用**

### EIP-4844（2024 年 3 月）

以太坊的重大升級，讓 L2 費用大幅下降：

```
升級前：摘要存入永久儲存（calldata）→ 貴
        Arbitrum 平均：$0.1 – $0.5 / 筆

升級後：摘要存入臨時 Blob（18 天後自動刪除）→ 便宜 10–100 倍
        Arbitrum 平均：$0.001 – $0.01 / 筆
```

**費用仍會波動：** L2 擁擠時，執行費上漲 + 提交主網頻率增加，雙重壓力。但整體仍比主網低幾個數量級，且持續在改善。

---

## 核心概念五：L3

L3 是建在 L2 上面的第三層，邏輯和 L2 一模一樣，只是錨定對象換成 L2：

```
L1（以太坊主網）← 安全性最高，最慢最貴
      ↑ 錨定
L2（Arbitrum、Base）← 快 10–100 倍，費用低
      ↑ 錨定
L3 ← 更快、更便宜、高度客製化
```

**L3 存在的理由：** 特定應用有特殊需求（遊戲的微型交易、企業私有鏈、特定 DeFi 協議），L2 無法滿足時用 L3 客製化。

**主要框架：** Arbitrum Orbit、OP Stack

## ✓ 學完能做到

**情境：** 評估兩個 agent payment 方案，方案 A 在 Arbitrum（Optimistic），方案 B 在 zkSync（ZK）：

- **日常付款：** 兩者費用相近且都很低，方案 A 預設相信每筆交易、7 天挑戰期確保安全；方案 B 靠數學證明即時確認
- **提款回主網：** 方案 B（zkSync）更方便，幾分鐘就能到帳；方案 A 需要等 7 天
- **為什麼都選 L2：** 主網每筆 Gas 費可能是付款金額的幾百倍，L2 讓微型付款在經濟上可行

---

## 與 Agent Payment 的連結

- **費用可行性：** 沒有 L2，agent 的微型付款根本不划算。EIP-4844 之後，$0.01 的付款配上 $0.001 的手續費，微型付款才真正可行
- **速度：** L2 的交易確認通常在幾秒內，agent 不需要等幾分鐘才知道付款成功
- **x402 選擇 Base（Coinbase 的 L2）的原因：** 費用極低、USDC 整合好、合規性相對清晰
- **L3 的未來：** 未來可能出現專為 agent 設計的 L3，交易費用接近零、針對高頻微型付款優化、內建身份驗證和授權機制
