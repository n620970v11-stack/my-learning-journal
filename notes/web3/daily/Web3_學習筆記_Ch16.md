# Web3 學習筆記｜第 16 章：隱私技術

> 學習目標：能評估一個「隱私方案」是真隱私還是行銷話術
> 完成日期：2026/06

---

## 核心問題

區塊鏈的公開透明是優勢，但也是問題——怎麼在不透露具體內容的情況下，還是能讓別人驗證「這件事是真的」？

---

## 核心概念一：匿名 ≠ 隱私

鏈上資料本身是匿名的，但不是無法追蹤的。

```
鏈上記錄的是：
「地址 0x1234...abcd 轉了 1 ETH 給 0x5678...efgh」

不是：
「Jane 轉了 1 ETH 給某某人」
```

**但匿名不等於隱私：**

```
只要有一個連結點把地址和你的真實身份連起來
→ 你所有的鏈上行為都變成透明的

常見的連結點：
→ 你曾經公開過你的地址
→ 你用這個地址從交易所出金（交易所有 KYC 資料）
→ 你跟某個認識你的人互動過
```

**鏈上分析（On-chain Analysis）：** Chainalysis、Elliptic 這類公司專門從已知地址出發，追蹤資金流向，把地址和真實身份連結起來——這也是各國政府打擊洗錢的主要工具。

---

## 核心概念二：零知識證明（Zero-Knowledge Proof，ZKP）

### 直覺理解

用色盲朋友的例子：

```
你有一顆紅球和一顆綠球
朋友看不出顏色差異
你想證明「這兩顆顏色不同」，但不想說哪顆是哪個顏色

做法：
→ 朋友把球藏到背後，隨機決定要不要換位置
→ 拿出來問你：「我有沒有換？」
→ 你看顏色就知道答案
→ 重複 100 次全部答對
→ 朋友確信兩顆球顏色不同，但完全不知道哪顆是紅的
```

**核心：能證明「你知道某件事」，但對方完全學不到那件事的內容。**

### 在 Web3 的應用

```
證明你有足夠的錢付款
→ 不需要透露帳戶餘額
→ 只證明「我的餘額 ≥ 付款金額」

證明你的年齡超過 18 歲
→ 不需要出示身份證
→ 只證明「我符合年齡條件」

證明你是某個 DAO 的成員
→ 不需要透露錢包地址
→ 只證明「我持有這個 NFT」
```

### 跟 ZK Rollup 的連結

ZK Rollup 用的正是 ZKP：

```
L2 向以太坊主網證明：
「我處理的 1000 筆交易都是正確的」
→ 不需要重新執行 1000 筆交易
→ 只驗證這份數學證明
```

## 核心概念三：混幣器與 Tornado Cash 案

### 混幣器是什麼

切斷「資金來源」和「資金去向」之間的連結：

```
很多人把錢存進混幣器合約
→ 合約把所有人的錢混在一起
→ 從另一個地址提款
→ 沒辦法追蹤這筆提款的錢原本來自哪裡
```

類比：100 個人各自倒一杯水進大水池，再各自舀出一杯水——無法說「這杯水原本是誰的」。

### Tornado Cash 的運作

```
存入 1 ETH → 拿到一張 ZKP「存款收據」
→ 收據證明「有人存了 1 ETH」，但不透露是誰
→ 之後用收據從任意地址提款 1 ETH
→ 沒有任何鏈上紀錄能連結存款地址和提款地址
```

### 2022 年制裁事件

美國財政部把 Tornado Cash 列為制裁目標——Web3 歷史上第一次制裁**智能合約本身**（而不是公司或個人）。

**理由：** 北韓駭客集團 Lazarus Group 用 Tornado Cash 洗白超過 $7 億美元的被盜資金。

### 核心爭議

```
政府立場：
混幣器系統性地破壞 AML（反洗錢）追蹤能力
→ 讓所有人（包括犯罪者）都能洗錢
→ 一個工具如果主要用途是洗錢，就有理由制裁它

Web3 社群立場：
混幣器只是一段程式碼
→ 制裁程式碼等於限制言論自由
→ 隱私是基本人權
→ 正當用途的人（記者、異見人士）也因此失去保護工具
→ 菜刀可以殺人，但我們不會禁止菜刀
```

### 制裁的現實限制

```
Tornado Cash 是去中心化的智能合約：
→ 沒有公司、沒有辦公室、沒有可以關掉的伺服器
→ 開發者被逮捕了，合約還是在跑
→ 制裁讓使用變成違法，但無法讓合約停止存在
```

## 核心概念四：隱私幣

以太坊預設透明，用混幣器才有隱私。隱私幣是直接在協議層把隱私內建進去。

### Zcash（ZEC）

用 ZKP 讓交易內容完全加密——金額、寄件人、收件人都可以隱藏，但礦工還是能驗證交易合法（沒有憑空造幣）。

### Monero（XMR）

用三個技術同時保護隱私，**每筆交易自動保護，不需要額外操作**：

**Ring Signature（環形簽名）——隱藏寄件人**
```
發出一筆交易
→ 自動混入其他 10 個人的交易作掩護
→ 外人只知道「這 11 個地址裡有一個發出了這筆錢」
→ 無法判斷是哪一個
```

**Stealth Address（隱身地址）——隱藏收件人**
```
每次有人付款給你
→ 自動產生一個全新的一次性地址
→ 外人看不出這些地址都屬於同一個人
```

**RingCT——隱藏金額**
```
外人只能看到「有一筆交易發生了」
→ 看不到轉了多少錢、從哪裡來、到哪裡去
```

### Monero vs Tornado Cash

| | Tornado Cash | Monero |
|--|-------------|--------|
| 隱私方式 | 需要額外操作 | 協議預設隱私 |
| 基礎鏈 | 以太坊（預設透明） | 獨立鏈（預設隱私） |
| 監管結果 | 合約被制裁 | 大量交易所下架 |

### 隱私幣的現實處境

```
政府無法直接關掉 Monero
→ 改從「入口」和「出口」下手：

入口（On-ramp）：要求交易所不得上架 Monero
出口（Off-ramp）：要求交易所不得支援 Monero 換回法幣
→ 不能讓 Monero 消失，但大幅降低它的實用性
```

日本、韓國、澳洲等國明確禁止交易所上架隱私幣。

## 核心觀念：隱私技術的根本困境

> **技術越成功地保護隱私，就越難跟現有的監管體系共存。**

```
Monero 讓「提供資料」從技術上就不可能
→ 政府要求交易所提供資料也拿不出來
→ 不是不願意配合，是根本沒有這些資料
```

## ✓ 學完能做到

**情境：** 有人說「我用 Monero 付款給 agent，這樣最隱私、最安全」

```
正確回應：
技術上 Monero 確實隱私性很強，但：

第一：監管風險
→ 很多國家和交易所已禁止或下架 Monero
→ 企業客戶沒辦法接受這個合規風險

第二：流動性問題
→ 服務提供商不一定願意接受 Monero
→ 換回法幣的管道受限

第三：與業界方向相反
→ 目前 agent payment 選擇合規優先、用 USDC
→ 隱私和合規之間，業界選擇了合規
```

---

## 與 Agent Payment 的連結

- **ZKP 的未來應用：** 未來 agent 可能用 ZKP 證明「我有足夠預算」而不透露確切金額，在隱私和可驗證性之間取得平衡
- **合規優先：** 目前 agent payment（x402、Skyfire）選擇 USDC + 合規 L2，而不是隱私技術——企業客戶的 AML 合規需求高於隱私需求
- **隱私 vs 合規的張力：** Agent 需要一定程度的隱私（不讓競爭對手看到所有行為），但又需要合規——這個平衡點未來還會繼續演變
