## Key Takeaways / 重點筆記

### Module 1.6 — Context Window

**核心定義**
- Context Window = LLM 一次能處理的 token 總量上限
- **input + output 都算**在這個額度內（很重要：回答長度也會吃掉你的 context）
- 比喻：像一張有固定大小的工作桌，超過就放不下

**主流模型容量**
| 模型 | Context Window |
|------|---------------|
| Claude | 200K tokens |
| GPT-4 | 128K tokens |
| Gemini | 1M tokens |

**塞得進去 ≠ 有效**
- **Lost in the Middle 現象**：放在 context 中間的內容容易被模型忽略
- 模型對開頭與結尾的注意力較強，中段資訊容易被「遺忘」
- 結論：context 容量大不代表你可以無腦塞——位置很重要

**解決方案的伏筆（Module 3 RAG）**
- 既然塞整份文件不一定有效，就改成「只塞最相關的片段」
- RAG（Retrieval-Augmented Generation）= 先檢索 → 再生成
- 這是 Agent Manager 後續會深入的核心技術
