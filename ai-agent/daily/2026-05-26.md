## Key Takeaways / 重點筆記

### Module 1.4 — Token

- 電腦只認識數字，不認識文字，所以文字要先轉成數字才能處理
- **Token** = 文字被切割後的碎片，介於字母和完整單字之間
- **Tokenization（分詞）** = 把文字切成 token 的過程
- 常見演算法叫 **BPE（Byte Pair Encoding）**，切法是從大量資料中學出來的，不是人工定義
  - 直觀例子："playing" → ["play", "ing"]；頻率高的組合會合併成一個 token

**Token 計費與成本**

| 語言 | 估算 |
|------|------|
| 英文 | 約 1 token ≈ 4 個字元 |
| 中文 | 1 個中文字 ≈ 1.5～2 tokens（比英文貴） |

- LLM 按 token 收費，token 越多越貴、越慢
- **設計中文 Agent 時成本估算要特別注意**，同樣的內容中文 token 數會比英文多

**Context Window（上下文視窗）**

- LLM 一次能「看見」的 token 數量有上限
- 超過上限 → 模型開始「忘記」對話最前面的內容
- 常見上限：Claude Sonnet 200k tokens、GPT-4 128k tokens
- **Agent Manager 實戰**：system prompt 越精簡越好，冗長的 prompt 直接拉高每次呼叫成本
