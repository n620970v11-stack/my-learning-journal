## Key Takeaways / 重點筆記

### 七個角色

以「用街口綁玉山帳戶在全家買咖啡」為例：

| 角色 | 在本例中 | 核心功能 |
|------|----------|----------|
| Cardholder（持卡人） | 你 | 交易的發起人 |
| Merchant（商家） | 全家 | 交易的接收端，等待付款成功通知 |
| Payment Gateway（支付閘道） | 街口（前端收口） | 收集付款資訊、加密後送出 |
| Payment Processor（支付處理商） | 街口（後端傳話） | 路由授權請求，傳遞授權結果 |
| Issuer（發卡行/帳戶行） | 玉山銀行 | 管你的錢；授權時預扣，清算時撥款 |
| Card Network（卡組織） | Visa / Mastercard | 讓兩家不認識的銀行溝通；制定規則、傳遞訊息、仲裁爭議。**不保管錢** |
| Acquirer（收單行） | 華南銀行 | 幫全家收款；清算後把錢入帳給全家 |

> **街口同時扮演 Gateway + Processor**，業界稱為 PSP（Payment Service Provider）。所以本例看起來是六個角色，拆開才是七個。

---

### 授權（Authorization）— 按下付款後幾秒內

```
去程（付款請求）：你 → 街口 → 華南 → Visa → 玉山（最後決策：批准 or 拒絕）
回程（授權結果）：玉山 → Visa → 華南 → 街口 → 你的手機 + 全家收銀機
```

- 玉山是**最後一關**，它才是批准/拒絕的決策者，Visa 負責橋接華南和玉山
- **這個過程裡，錢還沒有移動**
- 玉山只是把那筆錢「貼標籤鎖住」，標記為「已承諾給全家」

---

### 清算（Settlement）— 1-2 天後

```
玉山 → Visa → 華南 → 全家入帳
```

- 到這一步，全家才真的拿到那杯咖啡的錢
- **刷卡成功 ≠ 商家拿到錢**，中間差了 1-2 天的結算期

---

### 為什麼每個角色都不可少

- 街口不見了 → 沒有介面跟銀行溝通，付款請求發不出去
- 玉山不見了 → 沒有人幫你管錢、確認餘額，授權無從發生
- Visa 不見了 → 玉山和華南互不認識，訊息傳不過去，錢也流不動
- 華南不見了 → 全家沒有收款銀行，就算授權通過也收不到錢

---

### Agent Payment 伏筆 ⚠️

這套七角色系統，每一層都假設**發起付款的是人類**：
- 玉山用你的 OTP 確認身份
- Visa 用 3DS 驗證你的意圖
- 華南假設商家端有人確認交易

**Week 6 的核心問題**：當付款人換成 AI Agent，哪幾層會直接失效？
